# 统一预测能力协议

> 日期：2026-05-08
> 范围：新增预测体系接入前的 capability 协议、注册表、执行边界与验收门禁。

## 目标

FateCat 后续会接入黄历/择日、六爻、梅花、奇门、大六壬、风水九星、姓名合婚等体系。所有新增体系必须先成为独立 `capability`，不得直接混入综合八字默认报告。

## 当前状态

| capability | 状态 | 默认可见性 | 说明 |
| --- | --- | --- | --- |
| `bazi` | `production` | `default` | 唯一默认生产能力，走综合八字纯分析链路。 |
| `almanac` | `production` | `standalone` | 黄历/择日独立能力，复用 lunar-python 宜忌、冲煞与吉凶神煞数据，不混入默认报告。 |

## 资产边界

- `assets/fate/capabilities/registry.json`：能力注册真相源。
- `assets/fate/capabilities/schemas/`：协议、输入、输出、证据字段说明。
- `assets/fate/capabilities/profiles/`：各能力独立报告 profile。
- `modules/fate_core/src/fate_core/capabilities/`：运行时加载、校验和执行路由。

## 执行规则

- `defaultVisibility=default` 必须且只能用于 `bazi`。
- `planned` / `experimental` 能力不得执行，只能登记。
- 每个能力必须声明 input、engine、report、evidence、riskPolicy。
- 每个能力必须声明免责声明与禁止断语。
- Web/API/Bot 后续只能通过统一 executor 调用能力，不允许前端自行拼接报告块。

## CLI 验收入口

```bash
cd scripts/project
.venv/bin/python -m fate_core.cli capabilities --pretty
.venv/bin/python -m fate_core.cli capability bazi --input-json '{"birthDateTime":"1990-01-01 08:00:00","gender":"男","longitude":116.4074,"latitude":39.9042,"birthPlace":"北京"}'
.venv/bin/python -m fate_core.cli capability almanac --input-json '{"dateRange":{"start":"2026-05-08","end":"2026-05-10"},"eventType":"出行","place":"北京"}'
.venv/bin/python -m fate_core.cli capability ziwei --input-json '{"birthDateTime":"1990-01-01 08:00:00","gender":"男","longitude":116.4074,"latitude":39.9042,"birthPlace":"北京"}'
.venv/bin/python -m fate_core.cli capability meihua --input-json '{"question":"测试问题能否推进","castMethod":"number","castValue":"3,8,6"}'
```

`liuyao` 等 planned 能力调用必须失败并提示尚未生产化。`almanac`、`ziwei` 与 `meihua` 已进入生产 capability，但仍是 standalone；Web 默认 Markdown 报告不得自动输出黄历择日、紫微或梅花块。

## 上线流程

1. 在 `registry.json` 中登记 capability，初始状态必须是 `planned`。
2. 补 profile 与 schema 测试，确认不进入默认 `bazi` Markdown。
3. 实现 provider/usecase，并补结构化 evidence。
4. 补 Web/API/Bot 独立入口和 Markdown 快照。
5. acceptance 通过后，才能把 status 改成 `production`。
