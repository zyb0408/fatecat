# Planning Summary
本任务按 100% scorecard -> 材料资源图 -> 基础排盘与时间边界 -> 规则来源矩阵 -> 高级格局 -> 合化成败 -> 用神裁决 -> 岁运专题 -> benchmark/golden -> 报告证据 -> 维护性 -> 最终 review 的顺序推进。

# Lifecycle Gates
- SPEC：100% 定义、非目标、反证和材料边界已写入任务文档
- PLAN：TREE_SPEC.json 渲染成可执行 TP 树，ready queue 明确
- BUILD：按 wave 执行叶子节点，不跨包混改
- TEST：每个叶子节点有本地命令或可审查产物
- REVIEW：关键能力面跑 auto-review / principle gate / policy tests
- SHIP：只在 P0/P1 完成且本地 gate 通过时关闭任务；命中率未达标则保留 WARN 不强宣 100%
- 禁止跳过未闭合 gate 直接前进

# Simplest Path
先冻结当前 checkpoint，再把 100% 拆成可执行 scorecard；优先补材料、规则和评测门禁，再做引擎实现；用失败最多的 MingLi 分类指导专题优先级，避免平均用力。

# Split Strategy
按能力维度拆顶层包；每个顶层包拆成 source/implementation/golden/benchmark/report 四类叶子；执行波次先打地基，再实现规则，再接 benchmark 和报告，最后审查。
