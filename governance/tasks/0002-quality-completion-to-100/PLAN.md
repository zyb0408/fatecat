# Planning Summary
本任务按七个质量维度拆成八个顶层包：先修任务控制面，再补本地开发/仓库/CI，再补八字基础和专业规则，再处理生产 HITL 和长期维护，最后审查收口。

# Lifecycle Gates
- SPEC：目标终态、真实约束、惯性约束、反证条件已记录
- PLAN：递归任务树、依赖、执行波次已由工具渲染
- BUILD：只做当前 ready 叶子节点，不混入无关重构
- TEST：每个叶子节点记录真实验证结果
- REVIEW：维度完成后执行对应 review lens
- SHIP：所有本地节点 Done 且 HITL 状态真实可解释后才交付
- 不得跳过 gate；未闭合当前 gate 前不得进入下一阶段或宣称 100%

# Simplest Path
先完成 TP-00 控制面修复；随后并行推进本地开发、结构、CI 文档化和轻量校验；再做八字基础/专业能力的规则证据扩展；最后处理生产 HITL 和全仓 closeout。

# Split Strategy
按质量维度拆顶层包，按可直接执行的验证路径拆叶子节点；生产公网验收作为单独 HITL 叶子，避免阻塞本地可完成任务；八字专业能力按格局、合化、用神、岁运四个专业能力面拆分。
