# Planning Summary
按生产服务地基优先级拆分：先锁节气/报告结构，再补 evidence 与边界治理，然后统一输入隐私和多端输出，最后补典籍引用、质量门禁与服务化文档。
- 编译节点总数: 35
- 叶子执行项: 25
- 执行波次数: 11
- 当前任务必须遵守 `SPEC -> PLAN -> BUILD -> TEST -> REVIEW -> SHIP`

# Lifecycle Gates
- SPEC：综合八字范围、非目标体系、输入输出和质量门禁写入任务文档。
- PLAN：完成 TP 任务树、依赖图、执行波次和验收清单。
- BUILD：按 execution wave 推进叶子任务，所有业务改动绑定 TP 编号。
- TEST：新增 pytest 与 acceptance 门禁覆盖核心路径。
- REVIEW：自审报告结构、隐私、证据链、导出包卫生和文档口径。
- SHIP：提交、推送、CI 通过，并生成 closeout 交付证据。
- 不得跳过未闭合 gate 直接进入下一阶段。

# Simplest Path
不新增预测体系，不重写历法算法；先把当前综合八字链路变成可回归、可审计、可多端一致的服务。

# Split Strategy
