# Agent Entry

## 项目工作协议

1. 不要在没有验证证据的情况下声明“已完成”或“已测试”。
2. 开始任务前先读取 `governance/INDEX.md`。
3. 根据 `governance/context/CONTEXT-ROUTER.md` 选择最小上下文。
4. 涉及架构边界时必须读取相关 ADR。
5. 涉及用户功能时必须产出 QA 计划或验证证据。
6. 高风险变更必须说明回滚路径。
7. 如果发现重复错误或标准缺失，记录到 `agent-governance/agent-feedback/`。
