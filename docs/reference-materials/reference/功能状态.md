# 功能状态（标准报告 / 未来功能）

> 目的：明确当前生产报告真实输出范围，避免“隐藏但可开”的旧口径继续误导使用者或 agent。

## 标准报告原则

- 标准报告默认输出已经产品化、测试覆盖、能在 Bot/API smoke 中稳定生成的综合八字内容：八字主线 + 袁天罡称骨民俗辅助。
- 紫微斗数保留为独立体系，必须通过 `report_system` / Web 控件 / API `options.reportSystem` / Bot 确认页按钮或 `ziwei` capability 单独选择，不再与综合八字混排。
- 黄历/择日已作为 `almanac` 独立 capability 提供 CLI/API 结构化输出，梅花易数已作为 `meihua` 独立 capability 提供轻量起卦盘面；二者仍不属于标准 Markdown 报告体系；六爻、奇门、大六壬、风水九星、姓名合婚等当前仍为 `planned`，未实现前不得进入综合八字默认报告。
- Web/Bot/Markdown 展示层不回显非北京类真实出生地区；真实地点只用于经纬度解析、后端计算和受控记录。
- 机器可读 `analysisEvidence` 默认隐藏，只进入 JSON/API 审计字段；Markdown 正文不增加“技术区”。
- `hide` 参数仍会传给遗留计算器，用于关闭待新功能化的扩展计算；它不再表示 TXT 报告里有隐藏扩展块可打开。
- 从标准报告退役的能力统一登记在 `contracts/fate/future_features.json`，后续必须以新功能重新设计输入契约、输出模板与测试。

## 修改指引

1. 改标准报告结构：只改 `domains/experience-delivery/services/fatecat-delivery/src/report_generator.py` 的 `generate_full_report()` / 独立体系生成函数，并同步回归测试。
2. 新增未来功能：先改 `contracts/fate/future_features.json` 或新增独立 profile，再补 usecase / API / Bot / 报告模板。
3. 改机器可读依据：先改 `contracts/fate/evidence_schema.json` / `weight_policy.json` / `classics_rule_index.json`，再改计算器 evidence mapper。
4. 禁止把未来功能重新塞回“隐藏块”；每个候选能力必须有独立输入契约和验收测试。
