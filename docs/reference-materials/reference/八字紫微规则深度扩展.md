# 八字紫微规则深度扩展

## 定位

本阶段把八字与紫微从“结构化排盘 + 标杆字段”继续推进到“规则应用可审计”。规则不写散在前端，也不复制商业 App 文案；所有新增解释深度先进入 `contracts/fate/rule_depth_registry.json`，再由后端 usecase 装配为 `baziRuleDepth` 与 `ziweiRuleDepth`。

## 设计边界

- 规则 registry 只保存规则 ID、条件、证据字段、权重、冲突策略、风险边界和来源规则 ID。
- 八字只消费已有综合八字结构化字段：`baziBenchmark`、`accuracyGuards`、`geju`、`yongShen`、`wuxingScores`、`majorFortune` 等。
- 紫微只消费 iztro/FateCat 结构化字段：`palaceAnalysis`、`ziweiStarTaxonomy`、`ziweiMutagenFlow`、`ziweiPatternMatches`、`ziweiPalaceTopics` 等。
- Web 工作台只展示后端结果，不重新实现命理规则。
- 神煞、称骨、健康、财富、婚恋等内容保持风险边界，不输出确定未来或专业替代建议。

## 已落地字段

| 字段 | 体系 | 说明 |
| --- | --- | --- |
| `baziRuleDepth` | 综合八字 | 日主强弱、格局、用神、干支关系、岁运触发、辅助边界的规则应用层 |
| `ziweiRuleDepth` | 紫微斗数 | 星曜亮度、三方四正、四化作用域、格局候选、十二宫专题、运限联动的规则应用层 |
| `analysisEvidence.items.baziRuleDepth` | 综合八字 | 将规则应用层关联回 `classics_rule_index.json` |
| `analysisEvidence.items.ruleDepth` | 紫微斗数 | 将紫微规则深度层关联回 `classics_rule_index.json` |
| `conflictResolution` | 共同 | 按 core/dynamic/topic/boundary 层级、置信度和权重生成解释优先级，并给出每组冲突的裁决解释 |
| `weightProfile` | 共同 | 汇总总权重、加权置信度、分层权重和 top rules，用于审计解释优先级 |
| `combinationStatements` | 共同 | 输出克制组合断语；每条必须携带规则 ID、证据字段、置信度和风险边界 |
| `narrativeSummary` | 共同 | 从 `combinationStatements` 和 `conflictResolution` 生成 Markdown 摘要；报告层只呈现，不重新计算命理规则 |
