# 八字紫微能力基线与缺口矩阵

> 任务归属：`0002-bazi-ziwei-benchmark-hardening / TP-01.02 / TP-01.03`

> 用途：固化当前 `bazi` 与 `ziwei` 的真实能力基线，并把缺口映射到 `0002` 任务树。本文是研发对照表，不把未实现能力声明为 production。

## 5. 总体完成定义

- `bazi` 与 `ziwei` 仍是两个独立体系，默认报告只允许 `bazi`。
- 所有新增 P0 字段都有 JSON 结构、evidence 规则 ID 和测试。
- Web 只消费后端结构化结果，不自写命理判断。
- acceptance 与 GitHub Actions 覆盖最终提交。
